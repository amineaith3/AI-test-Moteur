\section{Executive Summary}
This project details an end-to-end Data Science pipeline developed to estimate the fair market value of used cars in Morocco. By transitioning from automated web scraping to complex mathematical modeling utilizing Random Forest regressors and Target Encoding, we successfully engineered an Artificial Intelligence model capable of predicting prices with an elite \textbf{87.3\% accuracy} and an error margin of \textbf{$\pm$16,390 MAD}.

\section{Iterative Pipeline Evolution}

\subsection{Step A: Data Collection}
A custom Python scraper using \texttt{BeautifulSoup} was deployed to extract data from \texttt{moteur.ma}. The script paginated through over 500 pages, compiling a robust dataset of over 15,000 distinct listings.

\subsection{Step B: Exploratory Data Analysis}
Data visualization revealed crucial market dynamics:
\begin{itemize}
    \item \textbf{Depreciation Curve:} Prices drop significantly prior to 150,000 km.
    \item \textbf{Categorical Premiums:} Automatic transmissions command substantial premiums in the Moroccan market.
\end{itemize}

\subsection{Step C \& D: Modeling Iterations}
The pipeline evolved through four significant iterations:
\begin{enumerate}
    \item \textbf{Version 1:} Base model suffered from a 52,000 MAD Mean Absolute Error (MAE) due to only analyzing the parent \texttt{Brand}.
    \item \textbf{Version 2:} Extracted specific \texttt{CarModel} parameters. The precision increase dropped the error to 39,000 MAD.
    \item \textbf{Version 3:} Implemented \textbf{Target Encoding} to replace arbitrary boolean logic with mathematical regional devaluation baselines (e.g., Casablanca vs. El Jadida). Accuracy surged to 76.6\%.
    \item \textbf{Version 4 (Final Engine):} Isolated the dataset into an \textbf{Economy Tier} ($<$ 300,000 MAD) to prevent extreme luxury outliers from inflating the MAE. The resulting Random Forest model scored \textbf{87.3\% accuracy} (16,390 MAD MAE).
\end{enumerate}

\section{Evaluation Metrics}
Rigorously evaluated using \textbf{5-Fold Cross Validation} and \textbf{Train-Validation-Test (70-15-15) Splits} to guarantee immunity against overfitting.

\section{Business Application}
The pipeline is serialized and wrapped in a responsive \textbf{Streamlit Web Application}, enabling live market valuations for end-users based on real historical data.
